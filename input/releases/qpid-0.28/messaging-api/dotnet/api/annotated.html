ISessionReceiverISessionReceiver interface defines the callback for users to supply. Once established this callback will receive all messages for all receivers defined by the current session. Users are expected not to 'fetch' or 'get' messages by any other means. Users must acknowledge() the Session's messages either in the callback function or by some other scheme.
